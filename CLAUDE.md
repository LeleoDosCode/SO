# CLAUDE.md - Guia do Repositório de Sistemas Operacionais

## 🎯 Objetivo do Projeto
Documentar de forma profissional e organizada o conteúdo da disciplina de Sistemas Operacionais, unindo teoria e prática.

## 🏗️ Arquitetura de Pastas
- `conteudo-do-curso/`: Teoria dividida por módulos.
- `exercicios/`: Laboratórios e estudos de caso.
- `documentos/`: Recursos externos e referências.

## ✍️ Padrões de Documentação
- **Linguagem:** Português do Brasil.
- **Formato:** Markdown (.md).
- **Estilo:** Uso de tabelas para comparativos, listas para características e blocos de código para comandos.
- **Imagens:** Devem ser referenciadas via caminhos relativos (se existirem).

## 🚀 Fluxo de Git
- **Commits:** Devem ser atômicos e descritivos (ex: `feat: add module 1`, `fix: update lab-virtualization`).
- **Identidade:** Commits realizados sob o usuário `LeleoDosCode`.

## 📚 Referências Base
- Tanenbaum, A. S. (Sistemas Operacionais Modernos).
- Silberschatz, A. (Fundamentos de Sistemas Operacionais).
